Artifact Parsing

Parse Windows artifacts

Convert binary Windows artifacts into readable, searchable formats.

Start with what you're investigating:

  • What programs ran and when? — Program Execution
  • What files did the user open or access? — User Activity
  • What security events occurred? — System Events
  • What file system changes happened? — File System
  • How did malware survive reboots? — Persistence & Autostart
  • What sites were visited or downloaded? — Browser & Network

Program Execution

Evidence of programs that ran on the system.

Ready

Parse Prefetch files and Amcache to determine what programs executed, when they ran, and how many times. Critical for establishing whether malware or tools were used.

Open category

User Activity

Evidence of files and folders the user interacted with.

Ready

Parse LNK shortcuts and Jump Lists to see what files were opened, from where, and when. Reveals user behavior and accessed resources, including deleted files and network shares.

Open category

System Events

Windows Event Log entries for security and system activity.

Ready

Parse EVTX event logs to review logon events, process creation, service installations, and security alerts. The backbone of most Windows investigations.

Open category

File System

Low-level file system metadata and change journals.

Ready

Parse the Master File Table ($MFT) and USN Journal to see every file that existed, when it was created, modified, or deleted—even if the file itself is gone.

Open category

Persistence & Autostart

Mechanisms attackers use to survive reboots.

Ready

Parse Registry Run keys, scheduled tasks, services, and other autostart locations to find how malware maintains access across reboots.

Open category

Browser & Network

Web browsing history, downloads, and cached data.

Ready

Parse browser databases to reconstruct browsing history, downloads, cached pages, and cookies. Useful for phishing investigations and tracking data exfiltration.
