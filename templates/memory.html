Memory Analysis

See what was running

Extract processes, network connections, and loaded code from RAM captures.

Source artifact: memory image (raw, crash dump, or LiME).

Memory analysis answers questions disk forensics can't:

  • What processes were running? — Including hidden or injected code
  • What network connections were active? — Even after they've closed
  • What commands were typed? — Command history from running shells
  • What was in memory but never saved to disk? — Malware, decrypted data, credentials
